Account pools

Multiple keys for the same provider — the router rotates to the next on a rate-limit/quota error, below the routing profiles. Failover between providers is now handled by profiles: see the routing banner at the top (pin/clear a profile there) and docs/profiles.md.

Profiles & switching rules

A profile is a named set of alias→target rewrites ("which model goes where"). The active profile is chosen by pin > error-shift > schedule > default (pin/clear at the top banner). Changes apply live — no restart.

Metered API token prices ($ / 1M in·out) — used for real per-response cost

Context fitting (trim to fit a smaller window on failover downshift)

Safety net, not steady-state compaction (that's the harness's job). When a request is routed to a model whose context window is smaller than the client's — a failover downshift (e.g. 1M → 256K) where the grown conversation no longer fits — the older turns are mechanically dropped to a stable checkpoint (tool_use/tool_result pairs kept intact) so it fits. If a backend still rejects a request as too long, the real window is learned and the request is hard-trimmed and retried. No summarizer, zero added latency on the normal path.

Per-model context window (glob → tokens) — order specific ids before broad globs
