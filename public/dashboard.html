Scheduled routing (dodge peak-priced models by time of day)

During each window, requests for the model glob are rewritten to a cheaper target. Times are in the timezone set per schedule — GLM peak = 14:00–18:00 at +08:00 (GLM-5.2 / GLM-5-Turbo cost 3× quota then). Evaluated against the system clock via UTC, so it's correct regardless of the host timezone.

Context compaction (summarize old turns to stay under the window)

When a request exceeds trigger% of the model's context window, older turns are summarized by the summarizer model into a compact checkpoint, landing around target% of the window (summary ≤ summary max% + a verbatim recent tail). If the summarizer fails, a deterministic mechanical trim to a safe checkpoint keeps the request valid. The per-session summary is cached, so the model is called only when (re)crossing the trigger.
